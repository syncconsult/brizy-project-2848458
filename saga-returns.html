The Saga Returns

The weekend of August 22-23, 2020, this issue once again appeared. You'll find a timeline of events and other stuff below.

The Issue

The current issue is still the same as last time. When working from the FireFox browser, this platform does not work. If I switch over to Chrome then it will sometimes be able to correct the mess that started from the Firefox creation/publish.

The Process

Here is a detailed walk through of the process. I'm attempting to build a website in Brizy Cloud, and then have it hosted on Render.

1) First I build the website on the Brizy Cloud platform. This process has always worked, and I've not had any issue with it.

2) After publishing the page inside of the editor, I then proceed back to the Dashboard and Publish the page to a third party hosting solution, Render.

2a) The necessary files are published to GitHub.

3) After publishing to GitHub, I go to Render and create a service. I choose the GitHub repository as the source.

4) Render then builds the site and publishes to their platform.

What Goes Wrong?

Sadly, a LOT goes wrong in this process.

In step 2/2a the files will not make it to GitHub if I use Firefox. I've been able to switch to Chrome and republish the files and most of the time they be published to GitHub, but not always.

In step 4, I often times get a "Can not connect to Brizy Database" error. Pretty self explanatory; something about the build has gone wrong, and it can't connect to the Brizy database.

More Issues

The idea behind Brizy is that you can make updates to the site via their editor, and then publish the changes to the web. I know how to build websites (see this one, this one or this one) but it's sometimes quicker to build a site in Brizy. The update portion, however, doesn't always go smoothly.

Since I'm unable to publish to GitHub from FireFox I have to go over to Chrome. Even then the changes don't always make it to GitHub on an update.
